## ゴール定義（このレベルになればOK）

* React の思想（宣言的UI / コンポーネント設計）を説明できる
* Hooks を適切に使い、状態・副作用を安全に扱える
* API通信・フォーム・一覧表示など典型的な画面を自力で実装できる
* コンポーネント責務分離・状態配置を意識できる
* 「次に何を学べば良いか」を自分で判断できる

## フェーズ1：React の基礎思想（最重要）

### 学習内容

1. Reactとは何か

   * UI = state の関数
   * 命令的 vs 宣言的
2. コンポーネント

   * 関数コンポーネント
   * JSX の考え方
3. props

   * 親 → 子のデータフロー
   * 再利用可能なコンポーネント

### 学習順序（おすすめ）

1. JSX の書き方
2. 関数コンポーネント
3. props
4. コンポーネント分割

### 到達目標

* 小さなUIをコンポーネントに分割できる
* propsの役割を説明できる

---

## フェーズ2：State とイベント（Reactの核）

### 学習内容

1. `useState`

   * state の考え方
   * 再レンダリング
2. イベントハンドリング

   * onClick / onChange
3. フォーム制御

   * controlled component

### 重点ポイント

* **state は「画面に影響する値」**
* 直接変更しない（immutable）

### 到達目標

* フォーム付きの簡単な画面を作れる
* なぜ state を使うか説明できる

---

## フェーズ3：副作用とライフサイクル

### 学習内容

1. `useEffect`

   * 副作用とは何か
   * 依存配列の意味
2. API通信（fetch / axios）
3. ローディング・エラー状態管理

### ここが初級最大の壁

* 無限ループ
* 依存配列の理解不足

### 到達目標

* APIからデータを取得して表示できる
* useEffect の動作を言語化できる

---

## フェーズ4：React × TypeScript（実務必須）

### 学習内容

1. props / state の型定義
2. イベント型
3. APIレスポンス型
4. Generics（最低限）

### ポイント

* **「全部厳密に」ではなく「壊れやすいところに型」**
* React.FC は無理に使わない

### 到達目標

* 型エラーを見て原因を理解できる
* propsの型を自力で設計できる
