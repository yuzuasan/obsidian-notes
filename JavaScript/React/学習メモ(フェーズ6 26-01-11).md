## 前提：Reactの再レンダリング

* state / props が変わると
  👉 **関数コンポーネントは上から下まで再実行**
* 「影響のある部分だけ実行」は **されない**
* 再レンダリング＝DOM即更新ではない
  （Virtual DOM の比較は別フェーズ）

---

## 重要概念：参照（reference）

* React は props を **===（参照）** で比較
* 毎回新しく作られるもの

  * 関数
  * オブジェクト / 配列
* number / string は値比較で変わりにくい

---

## useCallback

### 何をする？

* **関数の参照を固定する**

### 何のため？

* 子コンポーネントの **不要な再レンダリング防止**
* `React.memo` とセットで意味を持つ

### 使う判断基準

* 関数を props として子に渡している
* 子の再レンダリングが問題になっている

### 注意

* 処理を速くするものではない ❌
* 依存配列を間違えるとバグる

---

## useMemo

### 何をする？

* **計算結果（値）の参照を固定する**

### 何のため？

* 重い計算（filter / sort など）の **再実行防止**

### useCallbackとの違い

* useCallback：関数を覚える
* useMemo：**計算結果を覚える**

### 注意

* 軽い計算には不要
* とりあえず使うのは逆効果

---

## useRef

### 何をする？

* **値を保持するが、再レンダリングは起こさない**

### stateとの違い

|         | state | useRef |
| ------- | ----- | ------ |
| 値保持     | ○     | ○      |
| 再レンダリング | 起きる   | 起きない   |
| UI反映    | する    | しない    |

### 主な用途

* DOM参照
* 前回値の保持
* タイマーID
* 入力中の一時値

---

## カスタムHooks

### 正体

* Hooksを使える **ただの関数**
* JSXを持たない

### 目的

* **ロジックの再利用**
* UIとロジックの分離

### 判断基準

* 「この state / effect、他の画面でも同じ？」

  * YES → カスタムHooks
  * NO → コンポーネントに残す

### ポイント

* stateは共有されない
* 共有されるのは「ロジック」

---

## 最適化の順序（超重要）

1. 設計（責務の中心）
2. propsの流れ整理
3. それでも問題があれば

   * useCallback
   * useMemo

👉 **Hooksは魔法ではない**

---

## 一言まとめ（覚える用）

* Reactは毎回全部実行
* 参照が変わると再レンダリング
* useCallback＝関数
* useMemo＝値
* useRef＝再描画しない箱
* カスタムHooks＝ロジック共有
