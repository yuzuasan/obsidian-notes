## フェーズ1：要件定義の考え方（最重要）

### 目的

* 「言われたものを作る」から「本当に必要なものを定義できる」へ

### 学習テーマ

* 要件定義と設計の違い
* 業務要件 / 機能要件 / 非機能要件
* As-Is / To-Be
* 課題・目的・手段の切り分け

### ChatGPT学習モード活用例

**① インプット**

```
あなたは業務系SEの教育担当です。
要件定義の全体像を、初心者向けに体系的に説明してください。
```

**② 分解理解**

```
「業務要件」と「機能要件」の違いを、
実際の業務システム例で説明してください。
```

**③ 演習（重要）**

```
以下の顧客要望を要件に分解してください。
・目的
・業務要件
・機能要件
・非機能要件

【顧客要望】
Excelで管理している受注をシステム化したい
```

**④ フィードバック**

```
私の要件定義をレビューしてください。
不足点・曖昧な点・改善案を指摘してください。
```

## フェーズ2：顧客とのコミュニケーション

### 目的

* 技術説明ではなく「合意形成」ができるエンジニアになる

### 学習テーマ

* ヒアリングの型（5W1H / なぜを3回）
* 曖昧な要望の具体化
* ITが苦手な顧客への説明
* NGなエンジニア対応例

### ChatGPT学習モード活用例

**① ロールプレイ**

```
あなたは顧客役、私はSEです。
要件ヒアリングのロールプレイをしてください。
途中で曖昧な要望も出してください。
```

**② 振り返り**

```
今のヒアリングを評価してください。
良かった点・改善点・次に聞くべき質問を教えてください。
```

**③ 言い換え練習**

```
以下の技術的説明を、非ITの顧客向けに言い換えてください。
「バッチ処理で夜間にデータを集計します」
```

## フェーズ3：議事録・仕様確認

### 目的

* 「言った言わない」を防ぎ、信頼を積み上げる

### 学習テーマ

* 良い議事録の構成
* 決定事項 / 未決事項 / ToDo
* 仕様確認メールの書き方
* 曖昧表現の排除

### ChatGPT学習モード活用例

**① 議事録作成**

```
以下の打ち合わせ内容から、実務レベルの議事録を作成してください。
・決定事項
・宿題
・確認事項
```

**② レビュー**

```
この議事録でトラブルになりそうな点を指摘してください。
```

**③ 仕様確認メール**

```
顧客に送る仕様確認メールを作成してください。
前提知識がない相手でも理解できる文章にしてください。
```

## フェーズ4：見積りの基礎

### 目的

* 「勘」ではなく「説明できる見積り」を作る

### 学習テーマ

* 見積りの構成（工数 × 単価）
* WBSベースの見積り
* 要件不確定リスクの扱い
* 見積り説明の仕方

### ChatGPT学習モード活用例

**① 基礎理解**

```
業務システム開発における見積りの考え方を、
初心者向けに説明してください。
```

**② 演習**

```
以下の要件から簡易見積りを作成してください。
・画面数
・バッチ処理あり
・外部連携1本
```

**③ レビュー**

```
この見積りのリスクと、
顧客に説明すべき注意点を教えてください。
```

## フェーズ5：総合演習（疑似プロジェクト）

### 目的

* スキルを「使える状態」にする

### 内容

ChatGPTに**架空案件**を作ってもらい、以下を一気通貫で実施。

```
・要件ヒアリング（ロールプレイ）
・要件定義
・議事録作成
・仕様確認
・簡易見積り
```

**指示例**

```
中小企業向け販売管理システムの新規開発案件を作ってください。
私はSEとして対応します。
顧客役として進めてください。
```
