## 1. foreach とコレクション変更

- `foreach` 中に `List<T>` などのコレクションを変更すると **InvalidOperationException** が発生する
- 原因：
  - `foreach` は **Enumerator（列挙子）** を使う
  - `List<T>` は内部に **_version** を持っており、変更されると Enumerator が検知する
- 無限ループにはならない
- 実務での回避：
  1. ループ後にまとめて追加する ✅
  2. `for` ループを使う（境界条件に注意）
  3. `foreach` + `ToList()` にコピーする

## 2. async / await の挙動

```csharp
static async Task TestAsync()
{
    Console.WriteLine("A");
    await Task.Delay(1000);
    Console.WriteLine("B");
}

TestAsync();
Console.WriteLine("C");
```

* 出力順：`A → C → B`
* ポイント：

  * `async` メソッドは **await まで同期実行**
  * `await` で未完了 Task の場合、呼び出し元に制御が戻る
* UI アプリ（WPF / WinForms）では、`await` 後は **元の UI スレッドに復帰**
* サーバー側（ASP.NET Core）は SynchronizationContext がないので ThreadPool 上で再開
* 実務Tip：

  ```csharp
  await SomethingAsync().ConfigureAwait(false);
  ```

  * UI 復帰不要なライブラリで効率化

## 3. Entity Framework Core と Deferred Execution

```csharp
var query = context.Users.Where(u => u.IsActive).Select(u => new { u.Id, u.Name });
var list1 = query.ToList();
var list2 = query.Where(u => u.Name.StartsWith("A")).ToList();
```

* SQL 発行回数：2回

  1. `list1 = query.ToList()` → SQL1回
  2. `list2 = query.Where(...).ToList()` → SQL2回
* ポイント：

  * `IQueryable` は **Deferred Execution（遅延実行）**
  * `ToList()` を呼ぶと **一度メモリに取り込む**
  * `AsEnumerable()` はコピーせず、遅延実行は継続される

## 4. struct vs class（値型と参照型）

```csharp
struct Point { public int X; public int Y; }

Point p1 = new Point { X = 1, Y = 2 };
Point p2 = p1;
p2.X = 10;
Console.WriteLine(p1.X); // 1
```

* struct は **値型**

  * 代入でコピーされる → 元の値は変わらない
* class は **参照型**

  * 代入で参照を共有 → 元の値が変更される
* 値型の特徴：

  1. コピーしても元の値に影響しない ✅
  2. メソッドに渡すとコピーされる ✅
  3. null を直接代入できない ❌
  4. スタックに置かれることが多い ✅
* 実務Tip：大きい struct や変更頻繁な struct は避ける

## 5. ASP.NET Core × DI（依存性注入）

```csharp
builder.Services.AddSingleton<ICounter, Counter>();
```

* Singleton：アプリ全体で **1つのインスタンス** を共有
* 実行例：

  * `/test1` 呼び出し → Count = 1
  * `/test2` 呼び出し → Count = 2
* 実務ポイント：

  * 状態を持つ Singleton はスレッド安全に注意
* DI スコープまとめ：

  * Transient：依存解決のたびに新しいインスタンス
  * Scoped：**1リクエスト単位でインスタンスを共有**
  * Singleton：アプリ全体で1つ
