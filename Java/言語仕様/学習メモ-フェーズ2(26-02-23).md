# 🟢 フェーズ2：コレクションとジェネリクスの完全理解

## 1. コレクションフレームワーク

### List / Set / Map の特性

| 観点    | List    | Set   | Map        |
| ----- | ------- | ----- | ---------- |
| 順序    | あり（追加順） | 実装による | 実装による（キー順） |
| 重複    | あり      | なし    | キーはなし、値は可  |
| キーの有無 | なし      | なし    | あり（キーと値）   |

---

### Set 系

* **HashSet**

  * 内部：HashMap
  * 計算量：検索・追加・削除 O(1)（平均）
  * 順序保証：なし

* **LinkedHashSet**

  * 内部：HashMap + 双方向連結リスト
  * 計算量：O(1)（平均）
  * 順序保証：挿入順

* **TreeSet**

  * 内部：赤黒木（平衡二分探索木）
  * 計算量：O(log n)
  * 順序保証：ソート順（自然順 / Comparator）

---

### Map 系

#### HashMap

* 内部：ハッシュテーブル
* Java8以降：衝突が多いバケットは **赤黒木に変換**
* treeify 条件：

  1. バケット内要素数 ≥ 8
  2. テーブルサイズ ≥ 64
* 検索：`hashCode()` でバケット決定 → `equals()` で同一性判定
* equals/hashCode の契約を破るとデータ取得失敗

#### TreeMap

* 内部：赤黒木
* 計算量：O(log n)
* ソート順保持、範囲検索に有効

#### ConcurrentHashMap

* スレッドセーフで高性能
* Java7：Segment 単位ロック（分割ロック方式）
* Java8：Segment 廃止、バケット単位＋CASで同期
* synchronizedMap より並行性が高い

---

### ArrayList vs LinkedList

| 観点       | ArrayList  | LinkedList           |
| -------- | ---------- | -------------------- |
| 内部構造     | 配列         | 双方向連結リスト             |
| 末尾追加     | O(1)（償却）   | O(1)                 |
| ランダムアクセス | O(1)       | O(n)                 |
| 途中挿入     | 要素シフト O(n) | ノード操作 O(1) + 走査 O(n) |
| メモリ効率    | 良い         | 悪い                   |
| キャッシュ効率  | 良い         | 悪い                   |

* 実務では ArrayList が優先される

  * 走査・ランダムアクセスが多い
  * CPUキャッシュ効率良し
  * ノードオブジェクトが増えずメモリ効率も良い

## 2. ジェネリクス

### 型消去（Type Erasure）

* コンパイル時に型チェック
* 実行時には型情報が消える
* 型安全はコンパイル時に保証される

---

### ワイルドカード

| 型             | 追加 | 取得     | 用途             |
| ------------- | -- | ------ | -------------- |
| `List<T>`     | OK | T      | 両方             |
| `List<?>`     | ❌  | Object | 読み取り           |
| `? extends T` | ❌  | T      | Producer（読み取り） |
| `? super T`   | T  | Object | Consumer（書き込み） |

* `? extends T` → 「T のサブ型のどれか」
* `? super T` → 「T の親型のどれか」
* PECS 原則：**Producer Extends, Consumer Super**

---

### 実務的応用例

```java
void copy(List<? extends Number> src,
          List<? super Number> dest)
```

* `src` → Producer → `? extends Number` で安全に取り出せる
* `dest` → Consumer → `? super Number` で安全に書き込める
* PECS に従うことで型安全かつ柔軟なコピーが可能
