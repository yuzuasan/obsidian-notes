# フェーズ5：I/O・NIO・ファイル操作

## 1. Java I/O の基本構造

JavaのI/Oは **ストリームベース**で設計されている。

```text
データの流れ = Stream
```

主に2種類ある。

| 種類       | 対象      | 代表クラス                      |
| -------- | ------- | -------------------------- |
| バイトストリーム | バイナリデータ | InputStream / OutputStream |
| 文字ストリーム  | テキストデータ | Reader / Writer            |

---

# 2. InputStream / OutputStream（バイトストリーム）

バイナリデータを扱う。

```text
byte単位で読み書き
```

使用例

* 画像
* 動画
* PDF
* ZIP
* バイナリファイル

### 代表クラス

```text
FileInputStream
FileOutputStream
BufferedInputStream
BufferedOutputStream
```

例

```java
InputStream in = new FileInputStream("image.png");
```

---

# 3. Reader / Writer（文字ストリーム）

テキストデータを扱う。

```text
char単位で読み書き
```

使用例

* CSV
* JSON
* TXT
* XML

### 代表クラス

```text
FileReader
FileWriter
BufferedReader
BufferedWriter
```

例

```java
Reader reader = new FileReader("data.txt");
```

---

# 4. バイト → 文字変換（InputStreamReader）

テキストファイルは実際には **バイト列**として保存されている。

そのため

```text
byte → char
```

変換が必要になる。

その役割を持つのが

```text
InputStreamReader
```

構造

```text
BufferedReader
     ↑
InputStreamReader
     ↑
FileInputStream
```

例

```java
BufferedReader reader =
    new BufferedReader(
        new InputStreamReader(
            new FileInputStream("data.txt"),
            StandardCharsets.UTF_8
        )
    );
```

---

# 5. FileReader の正体

`FileReader` は実際には

```text
InputStreamReader のラッパー
```

である。

内部的には

```text
FileInputStream + InputStreamReader
```

を使用している。

注意点

```text
OSのデフォルトエンコーディングを使用
```

そのため実務では

```text
FileReaderはあまり推奨されない
```

---

# 6. バッファリング（Buffered）

バッファリングの目的

```text
I/O回数削減 → 高速化
```

理由

```text
ディスクアクセスは遅い
```

通常のI/O

```text
1文字 → 1回I/O
```

Buffered使用

```text
大量データをまとめて読み込む
```

代表クラス

```text
BufferedReader
BufferedWriter
BufferedInputStream
BufferedOutputStream
```

---

# 7. NIO.2（Java7以降）

Java7で導入された **新しいファイルAPI**

主なクラス

```text
Path
Files
```

---

## Path

ファイルシステム上の **パス（場所）** を表す。

```java
Path path = Path.of("data.txt");
```

### File との違い

| File   | Path        |
| ------ | ----------- |
| 古いAPI  | 新しいAPI      |
| ファイル中心 | パス中心        |
| 操作が分散  | Filesクラスに集約 |

---

## Files

ファイル操作を提供するユーティリティクラス。

代表API

```text
Files.readString()
Files.readAllLines()
Files.lines()
Files.write()
Files.exists()
Files.delete()
```

例

```java
List<String> lines =
    Files.readAllLines(path, StandardCharsets.UTF_8);
```

---

# 8. 大きいファイルの読み込み

方法の違い

| API          | メモリ |
| ------------ | --- |
| readString   | 大   |
| readAllLines | 大   |
| lines        | 小   |

巨大ファイル（ログなど）

```java
try (Stream<String> lines = Files.lines(path)) {
    lines.forEach(System.out::println);
}
```

特徴

```text
遅延読み込み（Stream）
```

---

# 9. シリアライズ

シリアライズとは

```text
オブジェクト → バイト列
```

に変換すること。

用途

* ファイル保存
* DB（BLOB）
* ネットワーク通信

---

## Serializable

シリアライズ可能にするには

```java
implements Serializable
```

を付ける。

```java
class User implements Serializable {
}
```

特徴

```text
マーカーインターフェース
（メソッドを持たない）
```

---

## ObjectStream

シリアライズ用ストリーム

| 処理   | クラス                |
| ---- | ------------------ |
| 書き込み | ObjectOutputStream |
| 読み込み | ObjectInputStream  |

例

```java
ObjectOutputStream oos =
    new ObjectOutputStream(
        new FileOutputStream("user.dat")
    );
```

---

# 10. transient

シリアライズ対象外にするフィールド

```java
class User implements Serializable {
    String name;
    transient int loginCount;
}
```

特徴

```text
復元時 → デフォルト値
```

| 型       | 値     |
| ------- | ----- |
| int     | 0     |
| boolean | false |
| Object  | null  |

---

# 11. serialVersionUID

クラスの **バージョン識別子**

```java
private static final long serialVersionUID = 1L;
```

役割

```text
保存時クラス
復元時クラス
の互換性チェック
```

書かない場合

```text
JVMが自動生成
```

問題

```text
クラス変更 → UID変更 → InvalidClassException
```

そのため

```text
実務では明示的に定義する
```

---

# 12. Serializable の問題点

Javaのネイティブシリアライズには問題がある。

```text
セキュリティリスク
可読性が低い
他言語互換なし
```

そのため実務では

```text
JSONシリアライズ
```

を使うことが多い。

代表ライブラリ

```text
Jackson
Gson
JSON-B
```

例（Jackson）

```java
ObjectMapper mapper = new ObjectMapper();

String json = mapper.writeValueAsString(user);
User user = mapper.readValue(json, User.class);
```

# フェーズ5まとめ

重要ポイント

```text
I/Oはストリームベース
byte系 → InputStream
文字系 → Reader
Buffered → 高速化
NIO.2 → Path / Files
Serializable → オブジェクト保存
serialVersionUID → クラス互換管理
```
