## 1. JOIN の基本

* **LEFT JOIN**：左テーブルの行は必ず残る。右テーブルにマッチがない場合は `NULL`。
* JOIN 条件は **ON** に書く。WHERE に書くと LEFT JOIN の特性が失われる場合がある。
* 実務でよくあるパターン：

  ```sql
  SELECT e.emp_id, e.name, d.dept
  FROM employees e
  LEFT JOIN departments d
    ON e.emp_id = d.emp_id AND d.dept = '開発部'
  WHERE e.emp_id = 2;
  ```

## 2. 集計関数

* 基本：

  * `COUNT(column)`：NULL を無視してカウント
  * `COUNT(*)`：行数すべてをカウント
  * `SUM(column)` / `AVG(column)` / `MAX(column)` / `MIN(column)`：NULL を無視
* **GROUP BY** でグループ化する
* **HAVING** で集計結果に条件をかける

  ```sql
  SELECT d.dept, COUNT(*) AS num_employees
  FROM employees e
  LEFT JOIN departments d
    ON e.emp_id = d.emp_id
  GROUP BY d.dept
  HAVING COUNT(*) >= 10;
  ```

## 3. NULL の扱い

* NULL は特別な値。「空文字」や 0 とは異なる
* 比較には `IS NULL` / `IS NOT NULL` を使用
* 集計関数や計算式では NULL の扱いに注意

  * `SUM(salary)` → NULL は無視
  * `salary + bonus` → どちらかが NULL だと結果も NULL
  * NULL を置き換える場合は `COALESCE(salary, 0)`

## 4. 文字列操作

* **LIKE**：あいまい検索

  * `%`：任意の文字列 0 文字以上
  * `_`：任意の 1 文字
* **CONCAT**：文字列結合（NULL があると結果も NULL になる場合あり）
* **TRIM / LTRIM / RTRIM**：余計な空白を除去
* **SUBSTRING / LEFT / RIGHT**：部分文字列抽出
* **NULL 対策**：

  ```sql
  SELECT CONCAT(COALESCE(first_name,''), ' ', COALESCE(last_name,'')) AS full_name
  FROM employees;

  SELECT CASE
           WHEN last_name IS NULL OR first_name IS NULL THEN '未登録'
           ELSE CONCAT(last_name, ' ', first_name)
         END AS full_name
  FROM employees;
  ```

## 5. 実務でよくあるハマりポイント

1. LEFT JOIN + WHERE で NULL 判定すると意図しない結果になる
2. 集計関数と計算式での NULL の扱いの違いを理解する
3. LIKE は大文字小文字や特殊文字に注意
4. CONCAT や文字列操作で NULL がある場合、COALESCE で置き換えると安全
