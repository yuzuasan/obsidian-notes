## 1. 日付集計 + HAVING
- WHERE は **明細レベルの条件**
- HAVING は **集計後のグループ条件**
- 例：1日の注文件数が10件以上
```sql
SELECT order_date, COUNT(*) AS order_count
FROM orders
WHERE order_date >= '2024-01-01'
  AND order_date < '2024-02-01'
GROUP BY order_date
HAVING COUNT(*) >= 10
ORDER BY order_date;
````

---

## 2. LEFT JOIN と COUNT の注意

* LEFT JOIN は「左側をすべて出す」
* COUNT(結合先のカラム) → NULL を除外
* COUNT(*) → NULL 行もカウント → 注文ゼロでも 1 になりやすい
* 業務SQLでは **顧客ごとの注文数は COUNT(o.order_id) が正解**
* 月数や種類数を数える場合

```sql
COUNT(DISTINCT DATE_FORMAT(o.order_date, '%Y-%m'))
```

---

## 3. サブクエリ（WHERE 句）

* サブクエリは「集合」として扱われる

```sql
SELECT *
FROM customers c
WHERE c.customer_id IN (
  SELECT o.customer_id
  FROM orders o
  WHERE o.cancel_flg = 0
);
```

* 内側：注文した顧客IDの集合
* 外側：その集合に含まれる顧客を抽出

---

## 4. IN と NULL の注意点

* `IN` のリストに NULL が含まれると、比較結果が UNKNOWN → フィルタされる
* 業務データに NULL がある場合、意図しない結果になることがある
* 覚え方：

  * IN → 集合に含まれるかだけ判定、NULLに弱い
  * EXISTS → 行が存在するか判定、NULL に強い

---

## 5. EXISTS の安全な使い方

```sql
SELECT *
FROM customers c
WHERE EXISTS (
  SELECT 1
  FROM orders o
  WHERE o.customer_id = c.customer_id
    AND o.cancel_flg = 0
);
```

* 「注文が1件でもある顧客」を安全に抽出
* NULL の影響を受けない
* 現場では IN より EXISTS が推奨

## 今日のまとめ

* LEFT JOIN + COUNT(*) は注意（注文ゼロでも 1 になりやすい）

* サブクエリは集合として理解
* IN は NULL に弱い、EXISTS は安全
